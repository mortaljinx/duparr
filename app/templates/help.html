HOW IT WORKS

⚡ RECOMMENDED ORDER OF OPERATIONS
1
SCAN LIBRARY * SCAN LIBRARY
Indexes all audio files into the database. Run this first. Subsequent scans are incremental — only new or changed files are processed. Does NOT run duplicate detection.
2
CLEAN COLLECTION TAGS ✦ CLEAN TAGS
Fixes albums where the tag contains collection path junk like "1989 - GREATEST HITS • Roy Orbison - 40 Greatest Hits". Extracts the real artist and album title and writes them to the files. Skippable if your library doesn't have these.
3
FIX TAGS ~ FIX TAGS
Fixes Album Artist tags on compilations and single-artist albums with featured guests. Smart detection — sets the correct artist, not just "Various Artists" for everything. Pitbull albums → Album Artist = Pitbull. Genuine compilations → Various Artists.
⚠ Run before deduplication — correct tags improve detection accuracy. Skip if not needed.
4
ALBUM DUPES @ ALBUM DUPES
Finds entire folders that are duplicate albums. Moves the weaker copy atomically — never leaves partial albums. Requires 3+ matched tracks. Ignores disc subfolders (CD1/CD2, Digital Media 01/02).
⚠ Must run BEFORE track-level dedup to avoid creating partial albums.
5
RESCAN ↺ RESCAN
Rescan after moving album duplicates to update the database before track-level dedup.
6
FIND DUPES ? FIND DUPES
Detects individual duplicate tracks. Groups by artist + title, scores each by quality. Results appear on the Dupes page.
7
REVIEW AND APPLY > APPLY ALL
Review duplicate groups on the Dupes page. Apply HIGH confidence first. Every move is fully undoable from the Undo page.
8
COVERS # COVERS
Finds and embeds missing album artwork. Tries MusicBrainz, Deezer, iTunes, and Last.fm in order. Run any time — works independently of the pipeline.

SAFETY

Nothing is ever deleted — files moved to /duplicates only
Every move logged to database
Full undo available from Undo page
Path traversal attacks blocked on all file operations
Same-folder tracks require fingerprint confirmation, not just metadata
Variants (remixes, live, acoustic) always preserved — never moved

TAG FIXER — FALSE POSITIVE PATTERNS

SINGLE ARTIST WITH FEATURED GUESTS
e.g. Calvin Harris — Motion. The smart dominant artist detection now handles these — sets Album Artist = Calvin Harris, not Various Artists. Watch for edge cases.
SPAM / WATERMARK TOKENS IN ARTIST FIELD
Some download collections embed site names (e.g. LanzamientosMp3.es) in the Artist tag. These are filtered out automatically when detecting the dominant artist.
COMPOSER CREDITS IN ARTIST FIELD
Old jazz or classical albums with composer names in the Artist tag rather than the performer. Use MusicBrainz Picard for these.
